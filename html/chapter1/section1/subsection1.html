One easy way to get started at programming in JavaScript is to interact with the JavaScript interpreter that is built into the browser you are using to view this page. We have set up the statements shown with a dark background such that you can click on them. The mouse click on JavaScript statements is programmed in such a way that a JavaScript interpreter is displayed, which can evaluate the statement and display the resulting value. By the way, the program that makes the mouse click on a JavaScript statement display the interpreter is itself written in JavaScript; we call it the script for the mouse click.

Expressions such as these, which contain other expressions as components, are called combinations. Combinations that are formed by an operator symbol in the middle, and operand expressions to the left and right of it, are called The value of an operator combination is obtained by applying the function specified by the operator to the arguments that are the values of the operands.

The convention of placing the operator between the operands is known as It follows the mathematical notation that the reader is most likely familiar with from school and everyday life. As in mathematics, operator combinations can be nested, that is, they can take arguments that themselves are operator combinations: \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} (3 * 5) + (10 - 6); \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=BQZgBAVGCsCUYGozAIwAYwFowDZYG4g}{\usebox\lstbox} As usual, parentheses are used to group operator combinations in order to avoid ambiguities. JavaScript also follows the usual conventions when parentheses are omitted; multiplication and division bind stronger than addition and subtraction. For example, \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} 3 * 5 + 10 / 2; \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=MwAgVCCsINQgjABhAehAJgNxA}{\usebox\lstbox} stands for \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} (3 * 5) + (10 / 2); \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=BQZgBAVGCsCUYGozAIwAYwHowCZYG4g}{\usebox\lstbox} We say that {\lstinline[mathescape=true]$*$} and {\lstinline[mathescape=true]$/$} have higher precedence than {\lstinline[mathescape=true]$+$} and {\lstinline[mathescape=true]$-$}. Sequences of additions and subtractions are read from left to right, as are sequences of multiplications and divisions. It is we humans who might get confused by still relatively simple expressions such as \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} 3 * 2 * (4 + (3 - 5)) + 10 * (27 / 6); \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=MwAgVCBM4gFALCA1HUBaEBWAlN5IBGABhlkgHYQB6EANmwG4g}{\usebox\lstbox} which the interpreter would readily evaluate to be 57. We can help ourselves by writing such an expression in the form \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} 3 * 2 * (4 + (3 - 5)) + 10 * (27 / 6); \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=MwAgVCBM4gFALCA1HUBaEBWAlNkAoFfARgAYZZIB2EAehADZsBuIA}{\usebox\lstbox} to visually separate the major components of the expression.

The interpreter always operates in the same basic cycle: It reads a statement from the browser, evaluates the statement, and prints the result. This mode of operation is often expressed by saying that the interpreter runs in a read-eval-print loop. Observe in particular that it is not necessary to explicitly instruct the interpreter to print the value of the statement.

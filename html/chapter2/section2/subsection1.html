2.2.1 Representing Sequences

\begin{figure}[H] \centering \maxsizebox{\linewidth}{0.8\paperheight}{\includegraphics[scale=0.8]{{img\string_javascript/ch2-Z-G-13}.svg}} \caption{The sequence 1, 2, 3, 4 represented as a chain of pairs. } \label{fig:sequence-of-pairs} \end{figure} One of the useful structures we can build with pairs is a sequence—an ordered collection of data objects. There are, of course, many ways to represent sequences in terms of pairs. One particularly straightforward representation is illustrated in figure~\ref{fig:sequence-of-pairs}, where the sequence 1, 2, 3, 4 is represented as a chain of pairs. The {\lstinline[mathescape=true]$head$} of each pair is the corresponding item in the chain, and the {\lstinline[mathescape=true]$tail$} of the pair is the next pair in the chain. The {\lstinline[mathescape=true]$tail$} of the final pair signals the end of the sequence by pointing to a distinguished value that is not a pair, represented in box-and-pointer diagrams as a diagonal line and in programs as the value of JavaScript's value {\lstinline[mathescape=true]$null$} . The entire sequence is constructed by nested {\lstinline[mathescape=true]$pair$} operations: \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} pair(1, pair(2, pair(3, pair(4, null)))); \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=A4QwlgTgFAjANAAgFANa0koCZErf9caAZlwPPw2gBZEA7AVwBsmBKd1gbiA}{\usebox\lstbox}

Such a sequence of pairs is called a list, and our JavaScript environment provides a primitive called {\lstinline[mathescape=true]$list$} to help in constructing lists.[1]

The above sequence could be produced by {\lstinline[mathescape=true]$list(1, 2, 3, 4)$}. In general, \begin{JavaScript} list(a$_{1}$, a$_{2}$, $\ldots$, a$_{n}$) \end{JavaScript} is equivalent to \begin{JavaScript} pair(a$_{1}$, pair(a$_{2}$, pair($\ldots$, pair(a$_{n}$, null)$\ldots$))) \end{JavaScript}

Our interpreter prints pairs using a textual representation of box-and-pointer diagrams. The result of {\lstinline[mathescape=true]$pair(1, 2)$} is printed as {\lstinline[mathescape=true]$[1, 2]$}, and the data object in figure~\ref{fig:sequence-of-pairs} is printed as {\lstinline[mathescape=true]$[1, [2, [3, [4, null]]]]$}:

\begin{lrbox}{\lstbox} \begin{JavaScriptClickable} const one_through_four = list(1, 2, 3, 4); \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=MYewdgzgLgBOCmB9KALATiArgcxYgZlmjALwwA2AltABQCMANDAExMDMTALAJQDcQA}{\usebox\lstbox}

We can think of {\lstinline[mathescape=true]$head$} as selecting the first item in the list, and of {\lstinline[mathescape=true]$tail$} as selecting the sublist consisting of all but the first item. Nested applications of {\lstinline[mathescape=true]$head$} and {\lstinline[mathescape=true]$tail$} can be used to extract the second, third, and subsequent items in the list. The constructor {\lstinline[mathescape=true]$pair$} makes a list like the original one, but with an additional item at the beginning. \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} head(one_through_four); // result: 1 \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=BYUwhgJgFA9gdiA+gF2AJxgVwObEQMyzQEoBuAKAHpKACNEAZ0wBtkAuGgRiA}{\usebox\lstbox} \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} tail(one_through_four); // result: [2, [3, [4, null]]] \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=C4QwlgNgFA9gdgUwPrABYCcYFcDmqkBm26AlANwBQA9FQAToIDOWEwAXLQNoBMANFwGZ+nACz84LCAF0ZQA}{\usebox\lstbox} \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} head(tail(one_through_four)); // result: 2 \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=BYUwhgJgFALmCWAbKB7AdiA+jYAnFArgObCYBmhuAlFQNwBQA9IwAS4gDOBiMAXCwCYgA}{\usebox\lstbox} \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} pair(10, one_through_four); // result: [10, [1, [2, [3, [4, null]]]]] \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=A4QwlgTgFAjADAGgAQHsB2BTA+gFwBYQoCuA5nlgGbEQCUA3AFAD0TSEGAzkQDY4BcSANrxkw0QCZRAZlEAWZGh7cAuqtVA}{\usebox\lstbox} \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} pair(5, one_through_four); // result: [5, [1, [2, [3, [4, null]]]]] \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=A4QwlgTgFArANAAgPYDsCmB9ALgCwkgVwHMcMAzQiASgG4AoAegYQjQGcCAbLALgQG14AgIyJ+AJjEBmMQBZEKLpwC6q1UA}{\usebox\lstbox}

The value {\lstinline[mathescape=true]$null$}, used to terminate the chain of pairs, can be thought of as a sequence of no elements, the empty list.

\subsubsection{ List operations

The use of pairs to represent sequences of elements as lists is accompanied by conventional programming techniques for manipulating lists by successively {\lstinline[mathescape=true]$tail$}ing down the lists. For example, the function {\lstinline[mathescape=true]$list_ref$} takes as arguments a list and a number $n$ and returns the $n$th item of the list. It is customary to number the elements of the list beginning with 0. The method for computing {\lstinline[mathescape=true]$list_ref$} is the following:

  • For $n=0$, {\lstinline[mathescape=true]$list_ref$} should return the {\lstinline[mathescape=true]$head$} of the list.
    • n
  • Otherwise, {\lstinline[mathescape=true]$list_ref$} should return the $(n-1)$st item of the {\lstinline[mathescape=true]$tail$} of the list.
\begin{lrbox}{\lstbox} \begin{JavaScriptClickable} function list_ref(items, n) { return n === 0 ? head(items) : list_ref(tail(items), n - 1); } \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=GYVwdgxgLglg9mABAGxgZygfQE4FNgAUMUuAtmgDSJgCUiA3gFCIuJ5QjZJIC8fiABmasRrAPyIAFrgCGAEyIlyNYaJEAuFOix5CUGTGSKyaGlSQBaRAEYaAbkYBfIA}{\usebox\lstbox} \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} const squares = list(1, 4, 9, 16, 25); list_ref(squares, 3); // result: 16 \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=MYewdgzgLgBBCOBXAhgJwKYRgXhgGwEtoAKARgBoYAWSgTktIDZKAmAVgEoBuAKEOgD6GAGbEEKDBEoBmbjwD08mJMR4oALhhMgA}{\usebox\lstbox}

Often we {\lstinline[mathescape=true]$tail$} down the whole list. To aid in this, our JavaScript environment includes a predicate {\lstinline[mathescape=true]$is_null$}, which tests whether its argument is the empty list. The function {\lstinline[mathescape=true]$length$}, which returns the number of items in a list, illustrates this typical pattern of use: \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} function length(items) { return is_null(items) ? 0 : 1 + length(tail(items)); } \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=GYVwdgxgLglg9mABAGwKZgOZQBYAoZSoC2AzgJSIDeAUInYgE6pQgNIwkD6YIyy+hUmVr1R9APyIADCLGiAXIgCMiANQp0WPFACGMfgWLkyAbmoBfIA}{\usebox\lstbox}

The {\lstinline[mathescape=true]$length$} function implements a simple recursive plan. The reduction step is:

  • The {\lstinline[mathescape=true]$length$} of any list is 1 plus the {\lstinline[mathescape=true]$length$} of the {\lstinline[mathescape=true]$tail$} of the list.

This is applied successively until we reach the base case:

  • The {\lstinline[mathescape=true]$length$} of the empty list is 0.

We could also compute {\lstinline[mathescape=true]$length$} in an iterative style: \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} function length(items) { function length_iter(a, count) { return is_null(a) ? count : length_iter(tail(a), count + 1); } \end{JavaScriptClickable} \end{lrbox} \begin{JavaScriptClickable} /*!\href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=GYVwdgxgLglg9mABAGwKZgOZQBYAoZSoC2AzgJSIDeAUInYqJLAiultgPoGoBOuAhgBpEEOOCgUa9aYh6ooIHkhgkOYEMmQCytGXvoB+EWLBRd+vQC5WmHF0J8o-GFv5lho8YgDUiAIxkANzmiAC+IXIKSjbs9rz4hKTCAAxB1KFAA}{\usebox\lstbox}!*/ return length_iter(items, 0); } \end{JavaScriptClickable}

\subsubsection{ Mapping over lists

One extremely useful operation is to apply some transformation to each element in a list and generate the list of results. For instance, the following function scales each number in a list by a given factor: \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} function scale_list(items, factor) { return is_null(items) ? null : pair(head(items) * factor, scale_list(tail(items), factor)); } \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=GYVwdgxgLglg9mABAZwgQwDYFMD6GbJQAUMUWAtsgDSLBrRwBOAlIgN4BQi3ijWUIRkgI4wIDBhJlKzLj3k8A-IjES5C+QC5EABzQxGRABZY0AEykVkrAFS16UJjXUbX3VJlz5CRKPsmkVsw0dAwszADcHAC+QA}{\usebox\lstbox}

We can abstract this general idea and capture it as a common pattern expressed as a higher-order function, just as in section~\ref{sec:higher-order-procedures}. The higher-order function here is called {\lstinline[mathescape=true]$map$}. The function {\lstinline[mathescape=true]$map$} takes as arguments a function of one argument and a list, and returns a list of the results produced by applying the function to each element in the list: \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} function map(fun, items) { return is_null(items) ? null : pair(fun(head(items)), map(fun, tail(items))); } \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=GYVwdgxgLglg9mABAWwIYAcAUowBpExQCmyAzgJSIDeAUIvYgE5FQiNIykD6YIANn0yESFOg3EMA-Il4CxE8QC5E6VDEbZwmABZFUAEyHEy5cvnkLL9NFhz4oawcJOmA3DQC+QA}{\usebox\lstbox} \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} map(abs, list(-10, 2.5, -11.6, 17)); \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=LYQwDgFCBGDOA0ACANgS1gFwgWgIwAYkAmAOgFYk9cSA2JXAdgEomBuIA}{\usebox\lstbox} \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} map(x => x * x, list(1, 2, 3, 4)); \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=LYQwDgFAHgBAvAPhrAVMgNDANgSwM4AuEAjJgEyYDMmALAJR0DcQA}{\usebox\lstbox}

Now we can give a new definition of {\lstinline[mathescape=true]$scale_list$} in terms of {\lstinline[mathescape=true]$map$}: \begin{lrbox}{\lstbox} \begin{JavaScriptClickable} function scale_list(items, factor) { return map(x => x * factor, items); } \end{JavaScriptClickable} \end{lrbox} \href{https://sourceacademy.nus.edu.sg/playground#chap=4&prgrm=GYVwdgxgLglg9mABAZwgQwDYFMD6GbJQAUMUWAtsgDSLBrRwBOAlIgN4BQi3ijWUIRknJoADkQAeiALwA+RFIBUtelCY1SFZMwDcHAL5A}{\usebox\lstbox}

The function {\lstinline[mathescape=true]$map$} is an important construct, not only because it captures a common pattern, but because it establishes a higher level of abstraction in dealing with lists. In the original definition of {\lstinline[mathescape=true]$scale_list$}, the recursive structure of the program draws attention to the element-by-element processing of the list. Defining {\lstinline[mathescape=true]$scale_list$} in terms of {\lstinline[mathescape=true]$map$} suppresses that level of detail and emphasizes that scaling transforms a list of elements to a list of results. The difference between the two definitions is not that the computer is performing a different process (it isn't) but that we think about the process differently. In effect, {\lstinline[mathescape=true]$map$} helps establish an abstraction barrier that isolates the implementation of functions that transform lists from the details of how the elements of the list are extracted and combined. Like the barriers shown in figure~\ref{fig:abstraction-barriers}, this abstraction gives us the flexibility to change the low-level details of how sequences are implemented, while preserving the conceptual framework of operations that transform sequences to sequences. section~\ref{sec:sequences-conventional-interfaces} expands on this use of sequences as a framework for organizing programs.
